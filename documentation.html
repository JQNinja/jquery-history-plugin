API Reference

There are 2 methods on jQuery.history object.

jQuery.history.init(callback [, options])

Initializes the plugin. You must call this method exactly once in jQuery(document).ready event. 

jQuery(document).ready(function($){

    $.history.init(function(hash){
        if(hash == "") {
            // initialize your app
        } else {
            // restore the state from hash
        }
    },
    { unescape: ",/" });
});

The first parameter is a callback function which is invoked with current hash when URL hash is changed. It is also invoked when a page is loaded so you can initialize your application in this function.

The second parameter is an option object. Currently only 'unescape' option is available to specify what characters are unescaped in URL hash. For example, if you want to keep the forward slash '/' from being escaped to '%2F', supply { unescape: "/" }.

Options
unescape false [default]The plugin encodes the URL hash using encodeURICompnent to escape the special characters which are "reserved" in RFC 2396.
trueThe plugin does not escape characters at all. This may cause a problem.
stringCharacters contained in the string are unescaped e.g. {unescape:"/;"} unescapes '/' and ';'.

jQuery.history.load(hash)

Loads a hash into location.hash, creates a new history entry, and invoke the callback function if necessary. Unless the provided hash is changed from current one the callback is not invoked. 

jQuery.history.load("new-hash-value");
